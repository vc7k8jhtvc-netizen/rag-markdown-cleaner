# RAG Markdown 批量清洗工具 — 使用说明

## 1. 工具做什么

批量清洗 OCR 产生的 Markdown：

- 扫描 `input/` 下 `.md`
- 自动分片
- 调用大模型 API 清洗
- 结果写入 `output/`
- 支持断点续跑、暂停、安全停止
- 支持 dry-run（只规划，不调用 API）

---

## 2. 目录结构

```text
rag-cleaner/                 # 项目根目录
├─ clean_auto/               # 程序代码
├─ input/                    # 待清洗 Markdown（你放这里）
├─ output/                   # 清洗结果
├─ logs/                     # 运行日志
├─ prompt.md                 # 系统提示词
├─ .env                      # API 配置（不要泄露）
├─ 一键菜单.bat              # Windows 一键启动（推荐）
├─ pyproject.toml
├─ requirements.txt
└─ 使用说明.md

3. 首次安装
3.1 进入项目根目录
cd "你的项目路径\rag-cleaner"
示例：

cd "D:\tools\rag-cleaner"
3.2 创建并激活虚拟环境
python -m venv .venv
.\.venv\Scripts\Activate.ps1
若被策略拦截：

Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\.venv\Scripts\Activate.ps1
3.3 安装
python -m pip install -U pip setuptools wheel
python -m pip install -e .
3.4 验证
python -m clean_auto --help
# 若已安装入口：
rag-cleaner --help
4. 配置
4.1 .env（项目根目录）
OPENAI_API_KEY=你的密钥
OPENAI_BASE_URL=https://你的接口地址/v1
OPENAI_MODEL=你的模型名
4.2 prompt.md
系统提示词。
不要用外层 ```markdown 代码块包裹；程序也会自动去掉外层代码块。

4.3 输入文件
把待处理 .md 放进：

input/
支持子目录。文件名以 _cleaned 结尾的会被跳过。

5. 日常使用（推荐）
方式 A：一键菜单（Windows 最简单）
打开项目根目录
双击 一键菜单.bat
选择：
选项	作用
1	Dry-run 预览（不调用 API）
2	试跑 1 个文件
3	正式处理全部
4	打开 input
5	打开 output
6	打开日志
0	退出
一键菜单.bat 会自动切换到自身所在目录，并优先使用 .venv\Scripts\python.exe。

方式 B：命令行
cd "你的项目路径\rag-cleaner"
.\.venv\Scripts\Activate.ps1

# 预览
python -m clean_auto --dry-run

# 试跑 1 个
python -m clean_auto --yes --max-files 1

# 全部
python -m clean_auto --yes
若已 pip install -e .：

rag-cleaner --dry-run
rag-cleaner --yes --max-files 1
rag-cleaner --yes
6. 常用参数
python -m clean_auto --help

python -m clean_auto --dry-run
python -m clean_auto --yes
python -m clean_auto --yes --max-files 3
python -m clean_auto --yes --strict
python -m clean_auto --yes --pause-between-files 2
python -m clean_auto --base-dir "你的项目路径\rag-cleaner" --dry-run
7. 暂停 / 停止 / 续跑
在项目根目录：

# 暂停
New-Item .\pause.flag -ItemType File

# 继续
Remove-Item .\pause.flag

# 安全停止
New-Item .\stop.flag -ItemType File
中断后再次运行同一命令即可续跑；已完成且指纹匹配的分片会跳过。

以下变更会导致重做：

源文件内容
prompt.md
模型名 / Base URL

10. 常见问题
找不到命令 rag-cleaner
.\.venv\Scripts\Activate.ps1
python -m pip install -e .
python -m clean_auto --help
pip 不是命令
python -m pip install -e .
input 里没有 Markdown
确认在项目根目录运行
确认文件在 input/，后缀是 .md
一键菜单乱码 / 选项炸行
请使用项目自带的英文版 一键菜单.bat（已避免 CMD 中文编码问题）。

换电脑后怎么用
拷贝整个项目文件夹
重新创建 .venv 并 pip install -e .
配置 .env
双击 一键菜单.bat
11. 最小工作流
1. 把 md 放进 input
2. 双击 一键菜单.bat
3. 选 1 预览
4. 选 2 试跑 1 个
5. 检查 output
6. 选 3 全部处理
12. 安全建议
不要分享 .env
大批量前先 dry-run，再试跑 1 个
重要原文自行备份
日志有基础脱敏，仍不建议公开 logs/
13. 一句话
进项目目录 → 文件放 input → 双击一键菜单 → 1 预览 → 2 试跑 → 3 全量 → 去 output 取结果。
