# RAG Markdown 批量清洗工具使用说明

本工具面向**中级注册安全工程师 AI 学习知识库**，用于批量清洗 PDF/OCR 转换后的 Markdown 教材。

主要处理：

- 明确的广告、销售和引流内容；
- 重复页眉、页脚和页码；
- PDF 转换产生的版式噪声；
- OCR 产生的明显格式错误；
- Markdown 标题、段落、列表和表格结构；
- 法律法规、标准规范、教材和真题内容。

程序会尽量保护教材正文、法条、标准编号、数字、单位、题目、选项、答案和解析。

---

## 1. 工具功能

程序会执行以下流程：

1. 扫描 `input/` 中的 Markdown 文件；
2. 读取文件并生成安全分片；
3. 调用 OpenAI Chat Completions 兼容接口；
4. 校验模型输出；
5. 检查正文是否被过度删除或异常扩写；
6. 保存分片结果和 metadata；
7. 所有分片成功后生成完整清洗文件；
8. 将需要人工复核的结果复制到 `review/`；
9. 支持暂停、安全停止和断点续跑。

主要功能包括：

- 支持 `input/` 子目录；
- 支持 dry-run；
- 支持限制单次处理文件数量；
- 支持 YAML Front Matter 严格校验；
- 支持模型上下文预算检查；
- 支持 HTTP 429 和服务器错误重试；
- 支持流式响应；
- 支持 partial 中断响应保存；
- 支持完整文件自动重新合并；
- 支持质量报告和人工复核目录；
- 支持 Windows 一键菜单。

## 5. 配置 API

### 5.1 创建本地配置文件

复制配置模板：

```powershell
Copy-Item .env.example .env
```

打开配置文件：

```powershell
notepad .env
```

### 5.2 填写基础配置

```dotenv
OPENAI_API_KEY=你的API密钥
OPENAI_BASE_URL=https://你的接口地址/v1
OPENAI_MODEL=你的模型名称
```

注意：

- 使用者必须填写自己的 API Key；
- 不要公开 `.env`；
- 不要把 API Key 写进源代码；
- 不要把 API Key 放进 Base URL；
- 已经泄露的 API Key 必须立即撤销并更换。

### 5.3 模型上下文预算

如果知道模型的上下文容量，可以配置：

```dotenv
OPENAI_CONTEXT_WINDOW=32768
OPENAI_MAX_OUTPUT_TOKENS=12000
OPENAI_TOKEN_PARAMETER=max_tokens
OPENAI_SAFETY_MARGIN_TOKENS=1024
```

部分模型可能使用：

```dotenv
OPENAI_TOKEN_PARAMETER=max_completion_tokens
```

如果不知道模型上下文容量，可以暂时使用：

```dotenv
OPENAI_CONTEXT_WINDOW=0
OPENAI_MAX_OUTPUT_TOKENS=0
OPENAI_TOKEN_PARAMETER=max_tokens
OPENAI_SAFETY_MARGIN_TOKENS=1024
```

`0` 表示不启用对应限制。

### 5.4 质量检查阈值

默认配置：

```dotenv
QUALITY_SEVERE_MIN_RETAINED_RATIO=0.30
QUALITY_WARNING_MIN_RETAINED_RATIO=0.50
QUALITY_REVIEW_MIN_RETAINED_RATIO=0.70

QUALITY_SEVERE_MAX_EXPANSION_RATIO=2.00
QUALITY_WARNING_MAX_EXPANSION_RATIO=1.50

QUALITY_HEADING_RETAINED_RATIO=0.50
QUALITY_QUESTION_RETAINED_RATIO=0.70
QUALITY_NUMBER_RETAINED_RATIO=0.70
QUALITY_TABLE_RETAINED_RATIO=0.50
```

普通用户建议先保持默认值。

修改 `.env` 后，需要重新启动程序，新的阈值才会生效。

## 8. 推荐运行流程

不要第一次就直接处理全部文件。

推荐顺序：

1. Dry-run；
2. 试跑一个文件；
3. 检查输出；
4. 小批量运行；
5. 最后全量处理。

### 8.1 Dry-run

Dry-run 只生成处理计划，不调用 API：

```powershell
.\.venv\Scripts\python.exe -m clean_auto --dry-run
```

重点检查：

- 找到的文件数量；
- 分片数量；
- 是否有规划失败文件；
- 是否存在超长代码块或表格；
- 默认分片字符数是否合适。

### 8.2 试跑一个文件

```powershell
.\.venv\Scripts\python.exe -m clean_auto --yes --max-files 1 --strict
```

### 8.3 小批量运行

例如处理最多 5 个待处理文件：

```powershell
.\.venv\Scripts\python.exe -m clean_auto --yes --max-files 5 --strict
```

### 8.4 全量运行

```powershell
.\.venv\Scripts\python.exe -m clean_auto --yes --strict
```

正式导入知识库前，建议始终使用：

```text
--strict
```


不要在运行中的程序尚未接收到停止请求时立即删除 `stop.flag`。

## 12. 输出结构

每个源文件会创建独立的输出目录：

```text
output/
└─ 教材名称_路径哈希/
   ├─ 教材名称_part_001_cleaned.md
   ├─ 教材名称_part_001_cleaned.md.meta.json
   ├─ 教材名称_part_002_cleaned.md
   ├─ 教材名称_part_002_cleaned.md.meta.json
   ├─ 教材名称_cleaned.md
   └─ 教材名称_cleaned.md.meta.json
```

文件说明：

- `*_part_XXX_cleaned.md`：模型清洗后的分片；
- `*.meta.json`：分片指纹、质量报告和状态；
- `*_cleaned.md`：全部成功分片合并后的完整文档；
- `*.partial.md`：流式请求中断时保存的部分响应。

完整文件丢失或 metadata 损坏后，再次运行程序会：

1. 验证已有分片；
2. 跳过有效分片；
3. 重新生成完整文件；
4. 不重复调用已完成分片的 API。

## 13. 人工复核目录

当完整文件被标记为：

```json
"review_required": true
```

程序会把复核副本写入：

```text
review/
```

示例：

```text
review/
└─ 教材名称_路径哈希/
   ├─ 教材名称_cleaned.md
   └─ 教材名称_review.json
```

复核报告会记录：

- 源文件；
- 输出哈希；
- 保留比例；
- 删除比例；
- 质量警告；
- 分片警告；
- 疑似新增网址；
- 疑似广告残留。

如果后续重新处理后不再需要复核，旧复核副本会被清理。

## 14. 质量检查

程序会检查：

- 输出是否为空；
- 是否发生严重截断；
- 是否异常扩写；
- 标题是否大量减少；
- 题目是否大量减少；
- 数字是否大量减少；
- 表格是否明显丢失；
- 是否新增原文不存在的网址；
- 是否仍然存在疑似广告或引流文字；
- YAML Front Matter 是否完整；
- 输出是否像 HTML 错误页或 API JSON 错误。

metadata 中：

```json
"review_required": false
```

表示程序没有发现明显风险。

```json
"review_required": true
```

表示结果已经保存，但需要人工检查。

自动质量检查不能证明模型输出完全正确。

## 15. 超长代码块和表格

如果单个围栏代码块或 Markdown 表格超过 `--max-chars`，程序会停止规划该文件，避免静默破坏结构。

可以先适当增大限制：

```powershell
.\.venv\Scripts\python.exe -m clean_auto `
  --dry-run `
  --max-chars 50000
```

如果仍然超限，建议人工拆分源文件中的超大代码块或表格。

单个文件规划失败不会拖垮整个批次，其他正常文件仍会继续处理。

## 16. 断点续跑

程序使用以下信息判断分片是否完成：

- 源文件 SHA-256；
- 分片 SHA-256；
- 提示词 SHA-256；
- 模型名称；
- API Base URL；
- 分片编号；
- 分片总数；
- 输出文件 SHA-256。

以下内容变化时，对应分片可能重新处理：

- 源文件内容；
- `prompt.md`；
- 模型名称；
- API Base URL；
- `--max-chars`；
- 分片算法；
- 分片数量。

## 17. 日志

运行日志位于：

```text

```

日志使用 JSONL 格式，每行一个 JSON 记录。

新日志只记录相对于项目目录的输出路径，不记录本机绝对路径。

日志会进行基础密钥脱敏，但仍不建议公开：

```text

```

历史日志如果由旧版本生成，仍可能包含本机绝对路径。

## 20. 常见问题

### 找不到 Python

检查：

```powershell
python --version
```

如果没有安装 Python，请先从 Python 官方网站安装。

### 找不到 `rag-cleaner`

建议直接使用：

```powershell
.\.venv\Scripts\python.exe -m clean_auto --help
```

如果需要重新安装：

```powershell
.\.venv\Scripts\python.exe -m pip install -e .
```

### 找不到输入文件

确认：

- 文件位于 `input/`；
- 文件扩展名是 `.md`；
- 文件名不是以 `_cleaned` 结尾；
- 当前项目目录正确。

### 程序一启动就停止

检查项目根目录是否仍有：

```text
stop.flag
```

确认旧任务已经退出后，通过一键菜单选项 `[7]` 清理控制标记。

### 程序一启动就暂停

检查是否存在：

```text
pause.flag
```

双击 `继续.bat`，或者执行：

```powershell
Remove-Item .\pause.flag -Force
```

### 接口返回 400

可能原因：

- 模型名称错误；
- Base URL 错误；
- 接口不支持某个 token 参数；
- 上下文超过模型限制。

如果接口不支持：

```text
max_tokens
```

可以在 `.env` 中改为：

```dotenv
OPENAI_TOKEN_PARAMETER=max_completion_tokens
```

如果两者都不支持：

```dotenv
OPENAI_MAX_OUTPUT_TOKENS=0
```

### 输出被标记为需要复核

打开：

```text
review/
```

重点检查：

- 法条；
- 标准编号；
- 年份；
- 数字和单位；
- 题目、选项、答案和解析；
- 表格；
- 广告是否漏删；
- 教材正文是否误删。
