# P0.1：自动续跑 PowerShell 行为测试任务

状态：待开发

风险：标准模式，测试补齐

依赖：无功能开发依赖

目标分支建议：`test/p0.1-powershell-auto-resume`

## 任务目标

为“开始处理全部文件”的既定自动续跑行为补齐 PowerShell 级测试，在后续菜单、manifest
和并发架构改造前建立稳定契约。本任务只新增或调整测试、测试夹具及必要的测试说明，不修改
生产运行逻辑、菜单产品语义或版本号。

## 范围

- 验证 `latest.json` 和目标 manifest 的发现、校验与分支选择行为。
- 验证自动续跑生成的 Python 命令参数和用户可见提示。
- 验证全量批次与部分文件选择批次的恢复范围。
- 验证项目 `.venv` 不可用时的既定失败行为，不允许静默使用系统 Python。
- 使用临时目录、伪造 manifest、stub Python 启动器和 mock 进程边界完成测试。

## 明确不在范围内

- 不实现或修改自动续跑功能，不改变菜单入口和现有提示语义。
- 不实现 P0.2 的心跳、SSE 接收量或终端刷新。
- 不实现独立分片执行单元、全局分片并发、实时面板、动态并发或 ETA。
- 不修改 `clean_auto/` 下运行逻辑，不修改版本号，不创建 Tag 或 Release。
- 不访问网络，不读取真实 `.env`、API Key、Token 或真实业务文件。

## 行为测试矩阵

| 场景 | 测试准备 | 预期行为 |
| --- | --- | --- |
| `latest.json` 不存在 | 临时 `logs` 目录无 latest 指针 | 不尝试恢复，进入既定的新全量批次命令路径。 |
| `latest.json` 损坏 | 写入无效 JSON | 按既定错误/回退契约处理，不拼接不可信参数，不崩溃到 PowerShell 堆栈。 |
| latest 指向的 manifest 不存在 | latest 使用虚构 batch ID | 按既定错误/回退契约处理，不执行不存在批次的恢复命令。 |
| manifest 损坏 | 写入无效 JSON | 拒绝恢复，不从损坏内容提取路径或参数。 |
| schema 不兼容 | 使用未知或不支持的 schema | 保守拒绝恢复并显示既定提示。 |
| 含 `pending` | 伪造最小合法 manifest | 调用 `--resume-batch`，保留 batch ID 与既定 `workers` 参数。 |
| 含 `interrupted` | 伪造最小合法 manifest | 调用 `--resume-batch`，显示自动继续提示。 |
| 含 `running` | 模拟上次异常退出遗留状态 | 作为可恢复任务处理，不隐式新建全量批次。 |
| 仅含 `failed` | 所有文件均为终态失败 | 不直接调用 `--resume-batch`；进入既定全量扫描路径以复用有效缓存。 |
| 无可恢复状态 | 全部为 `succeeded`/`skipped` | 创建新的全量处理批次。 |
| 部分选择批次可恢复 | manifest 仅包含选择文件 | 只恢复原选择范围，完成后不隐式启动全量批次。 |
| 全量批次可恢复 | manifest 表示全量范围 | 恢复原全量批次，不创建第二个批次。 |
| 项目 `.venv` 不可用 | 临时项目中不放置目标 Python | 显式失败并提示安装/修复，不静默调用系统 Python。 |

损坏和不兼容场景的精确预期应先从当前脚本行为固定为回归测试；若当前行为与路线中的产品
决策冲突，应停止该测试任务并单独提交产品缺陷，不得在测试 PR 中顺带修改运行逻辑。

## 测试隔离要求

- 每个用例使用测试框架提供的临时目录，所有 latest、manifest、选择文件和标志文件均为
  测试生成的最小伪造数据。
- PowerShell 子进程使用显式测试环境变量和 stub 可执行文件，不继承或枚举真实密钥。
- 测试不得加载仓库根目录或用户目录中的 `.env`，不得读取 `input/`、`output/`、`logs/`
  中的真实内容。
- 禁止真实 API 调用；命令执行边界应捕获参数、退出码、stdout 和 stderr。
- Windows 文本断言需覆盖 UTF-8/中文提示，并避免依赖机器用户名、绝对路径或当前时区。

## 建议落点

优先扩展现有 `tests/test_launcher.py` 的 PowerShell 静态/行为测试模式；仅当进程级行为测试使
该文件职责明显过重时，再新增名称明确的测试模块和 `tests/fixtures/` 下的最小夹具。生产脚本
只作为被测对象，不在本任务中修改。

## 验收标准

- 行为测试矩阵中的场景全部有可重复、互相隔离的自动化覆盖。
- 可恢复状态确实选择 `--resume-batch`；仅失败或已完成状态确实选择既定全量扫描路径。
- 部分选择批次不会越界处理其他文件；命令参数、退出码和关键中文提示均被验证。
- `.venv` 不可用时不会回退到系统 Python。
- 测试运行未读取真实 `.env`、密钥、Token、业务文件，也未发出网络请求。
- Windows 验收环境测试通过，现有完整测试与 Ruff 检查继续通过。
- PR diff 仅包含测试、测试夹具和必要测试文档；任何产品行为差异另开任务处理。

## 交付与复核

提交说明应列出覆盖的 manifest 状态、PowerShell 版本及实际执行的测试命令。评审重点检查
测试是否真正经过 PowerShell 行为边界、是否可能误读真实环境，以及断言是否锁定产品决策
而非实现细节。
